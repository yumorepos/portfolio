Polymarket Autopilot

Status: Live

Overview

Paper-trading research engine for prediction market strategies. Demonstrates quantitative mindset, engineering discipline, and product quality with end-to-end trading simulation.

Tech stack

Python 3.12+, FastAPI, SQLite, pytest (57 tests), Streamlit, mypy (strict typing), Ruff (linting)

How it works

  • Market Ingestion: Fetches live Polymarket data with offline demo mode for reproducible walkthroughs
  • Strategy Evaluation: Multiple strategies with benchmark-relative performance and deterministic ranking
  • Risk Management: Automated stop-loss (-10%) and take-profit (+20%) orders with position monitoring
  • Backtesting: Single-strategy replay with metrics (Sharpe, drawdown, win rate)
  • Dashboard: Streamlit interface with equity curves, strategy attribution, and open/closed positions

Why I built this

To showcase quantitative finance skills and systematic trading approach. This project demonstrates my ability to build production-grade financial systems with explicit risk controls, transparent analytics, and comprehensive testing—essential skills for data analyst and Python developer roles.

Interesting challenges

  • Deterministic Demo Mode: Created offline mode with seeded snapshots and trades for recruiter-safe walkthroughs without API dependencies
  • Strategy Attribution: Built explainability layer with deterministic rationale per strategy (return vs baseline, Sharpe, drawdown)
  • Risk Framework: Implemented automated stop-loss/take-profit with position monitoring cron jobs
  • Quality Gates: Achieved strict typing (mypy), linting (Ruff), and 57 passing tests with GitHub Actions CI
  • Production Readiness: Global error handling, structured logging, health checks (liveness/readiness)

What I'd improve next

  • Walk-forward evaluation: Rolling-window stability diagnostics
  • Execution realism: Configurable spread/slippage/latency models in backtests
  • Richer attribution: Reason-code decomposition at trade lifecycle level
  • Dataset tooling: Snapshot quality checks and scenario packs for stress testing

Key Features

  • CLI: 10+ commands (init, scan, trade, backtest, compare, report, monitor-positions)
  • Demo-ready: One-command reproducible demo (demo-run) for interviews
  • Testing: 57 tests covering core logic, strategies, risk controls
  • Documentation: Comprehensive README with architecture diagrams and command reference
  • CI/CD: Automated tests and lint checks on every push
