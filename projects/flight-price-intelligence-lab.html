Flight Price Intelligence Lab

Status: Live

Overview

Full-stack aviation analytics platform that transforms Bureau of Transportation Statistics data into actionable route intelligence with modern UI/UX.

Tech stack

Next.js 14, TypeScript, FastAPI, PostgreSQL, Recharts, Tailwind CSS, Vercel

How it works

  • Route Scoring: 0-100 scores blend fare attractiveness, reliability, and price stability
  • Deal Signals: Compare current fares against historical baselines with visual indicators
  • Interactive Visualizations: Recharts-powered trend analysis with orange gradient design system
  • Mock API Routes: Next.js API handlers for instant deployment without backend
  • Responsive Design: Mobile-first layouts with 87.5KB optimized bundle

Why I built this

To demonstrate full-stack development skills with a production-ready aviation analytics platform. It showcases modern frontend architecture (Next.js 14 + TypeScript), backend API design (FastAPI + PostgreSQL), DevOps maturity (CI/CD + automated testing), and a polished orange gradient design system.

Interesting challenges

  • Mock API Architecture: Designed Next.js API routes to serve demo data independently of backend deployment
  • Performance Optimization: Reduced bundle size to 87.5KB with sub-200ms load times
  • Design System: Created cohesive orange gradient theme (Orange-500 → Amber-600) matching FlightFinder branding
  • Production Hardening: Implemented global error handling, structured JSON logging, and security scanning (fixed 6 vulnerabilities)
  • Documentation Quality: Wrote 107KB of technical documentation including 7 challenge stories for interviews

What I'd improve next

  • Deploy backend to Railway/Fly.io for live data integration
  • Add E2E tests with Playwright or Cypress
  • Implement caching layer with Redis for performance
  • Add user authentication with NextAuth.js
  • Expand API coverage to more BTS datasets
